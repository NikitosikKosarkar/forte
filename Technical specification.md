# **ТЕХНИЧЕСКОЕ ЗАДАНИЕ**

Данное ТЗ содержит требования к разработке приложения-среды исполнения для работы с 4diac

## 1. Введение

Настоящее ТЗ содержит требования к среде исполнения для работы с 4diac (аналог FORTE).

## 2. Назначение разработки

Функциональное назначение: обеспечение возможности исполнения проектов в формате 4diac и взаимодействия со средой разработки.  
Эксплуатационное назначение: использование в учебном процессе для изучения принципов работы распределенных систем управления и стандарта IEC 61499.

## 3. Требования к программе или программному изделию

### 3.1. Требования к функциональным характеристикам

Программа должна обеспечивать выполнение следующих функций:

- Прием команд от среды разработки по протоколу, совместимому с 4diac, через TCP-соединение
- Создание и управление ресурсами, функциональными блоками и связями между ними
- Исполнение алгоритмов функциональных блоков при наступлении событий
- Загрузка начальной конфигурации из загрузочного файла (.fboot)
- Поддержка базовых типов функциональных блоков (E_RESTART, E_SR, STRING2STRING, OUT_ANY_CONSOLE)
- Обработка команд CREATE, WRITE, CREATE CONNECTION, START, QUERY

### 3.2. Требования к составу и параметрам технических средств

Приложение будет запускаться на персональных компьютерах под управлением ОС Linux или Windows

### 3.3. Требования к информационной и программной совместимости

Приложение будет совместимо с ОС Linux/Windows,
предполагает совместимость с протоколами взаимодействия среды 4diac
и загрузочными файлами формата .fboot

## 5. Требования к программной документации

Должно быть разработано руководство пользователя с описанием процедуры сборки и запуска среды исполнения для работы с 4diac

## 6. Стадии и этапы разработки

1. Определение требований
2. Проектирование архитектуры
3. Разработка сетевого модуля
4. Разработка ядра исполнения функциональных блоков
5. Тестирование
6. Документирование
